# Add Midtrans Payments

Use this skill when the user asks to add Midtrans, a payment gateway, VA, QRIS, GoPay, Dana, ShopeePay, or Indonesian checkout.

## Scope from prompt

Before coding, extract from the user's request:

1. **Payment flow** — one-time checkout, deposit, marketplace split, etc.
2. **Integration method** (default: **Snap Redirect** if unsure):
   - Snap popup — payment UI on your page (no redirect)
   - Snap Redirect — customer redirected to Midtrans-hosted page
   - Core API — custom UI with direct charge (credit card token, specific channel)
3. **Post-payment actions** — update order status, deduct stock, send email, etc.
4. **Lifecycle ops** — cancel, expire, refund, approve challenge (only if mentioned)
5. **Disbursement/payout** — Iris API (only if user explicitly asks; see [go.md](go.md) Iris section)

If the prompt is silent on integration method, use **Snap Redirect**.

If the user asks for subscriptions or recurring billing, explain that Midtrans has no native subscription product — implement manual recurring charges or ask which approach they prefer.

## Detect backend

Inspect the project and follow the matching guide:

| Signal | Guide |
|--------|-------|
| `composer.json`, Laravel, or PHP backend | [php.md](php.md) |
| `go.mod` or Go HTTP handlers | [go.md](go.md) |

If both exist, prefer the language of the order/payment module. If ambiguous, ask once.

## Steps

1. **Install the SDK** — see stack-specific file.

2. **Add environment variables**

   ```
   MIDTRANS_SERVER_KEY=SB-Mid-server-...
   MIDTRANS_CLIENT_KEY=SB-Mid-client-...
   MIDTRANS_IS_PRODUCTION=false
   ```

3. **Create a Midtrans config module** — initialize Server Key, sandbox/production flag, and client (Snap / Core API).

4. **Persist the order first** — save a unique `order_id` and integer `gross_amount` (IDR) in your database **before** calling Midtrans.

5. **Create a payment endpoint**

   - **Snap Redirect** (default): create transaction, return `redirect_url`.
   - **Snap popup**: return `snap_token`; load `snap.js` on frontend and call `snap.pay(token)`.
   - **Core API**: charge with `payment_type`; handle 3DS `redirect_url` on frontend if returned.

6. **Create a notification webhook** — `POST /payments/midtrans/notification` (or project convention):

   - Parse the notification payload.
   - **Always re-check status via API** (`Transaction::status` / `CheckTransaction`) — do not trust the payload alone.
   - Update order status idempotently (same notification may arrive more than once).

7. **Add result pages** — success, pending (VA/QRIS/e-wallet), and failure/cancel routes or API responses.

8. **Lifecycle endpoints (if in scope)** — cancel, expire, refund via Core API transaction actions.

9. **Frontend (Snap only)** — include Midtrans Snap JS:

   ```html
   <!-- Sandbox: keep .sandbox in URL. Production: use app.midtrans.com -->
   <script src="https://app.sandbox.midtrans.com/snap/snap.js"
           data-client-key="YOUR_CLIENT_KEY"></script>
   ```

## Status mapping

Map Midtrans status to your order model:

| Midtrans | Action |
|----------|--------|
| `pending` | `awaiting_payment` — common for VA, QRIS, e-wallet |
| `capture` + fraud `accept` | `paid` |
| `settlement` | `paid` (final for non-card channels) |
| `deny`, `cancel`, `expire` | `failed` or `cancelled` |
| `capture` + fraud `challenge` | `review_required` — approve/deny via Dashboard or API |

## Notes

- Amounts must be **integer IDR**. Convert foreign currency first; never round prematurely.
- `order_id` must be unique and generated by your system (e.g. `ORD-{uuid}`).
- On notification: verify via status API, then update DB idempotently.
- `pending` is normal for async channels — do not mark as failed immediately.
- Credit card: check both `transaction_status` and `fraud_status`.
- Sandbox uses `app.sandbox.midtrans.com`; production removes `.sandbox`.
- Local webhook testing: expose your server with ngrok or cloudflared.
- Never commit Server Key. Client Key is safe for frontend Snap only.
- Register the notification URL in [Midtrans Dashboard](https://dashboard.midtrans.com) or override per transaction.
